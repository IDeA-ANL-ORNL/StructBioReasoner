# Two-Step Tool Calling Implementation

## Overview

This document describes the implementation of the **two-step tool calling approach** that allows the LLM to actually call BindCraft during hypothesis generation.

This overcomes the OpenAI API limitation where you cannot use both `tools` and `response_format: json_object` simultaneously.

---

## The Problem

**OpenAI API Limitation**: You cannot use both:
- `tools` parameter (for function calling)
- `response_format: {type: "json_object"}` (for structured JSON output)

...at the same time.

**Our Need**:
- We NEED structured JSON to ensure hypotheses match our schema
- We WANT function calling to let LLM use computational tools
- **We can't have both in a single API call!**

---

## The Solution: Two-Step Approach

### Step 1: Generate Initial Hypothesis (Structured JSON)

```python
# Use response_format: json_object (no tools)
response_data = self.llm.generate_with_json_output(
    prompt,
    schema,
    system_prompt=system_prompt
)
```

**Result**: Structured hypothesis with literature/homology/de-novo sequences

### Step 2: Ask LLM if It Wants to Use Tools (Function Calling)

```python
# Use tools parameter (no json_object)
tool_response = self.llm.generate_with_tools(
    prompt=tool_decision_prompt,
    tools=tools,
    system_prompt="You are a computational biologist...",
    temperature=0.3
)
```

**Result**: LLM can make function calls to request tool execution

### Step 3: Execute Tools and Incorporate Results

```python
# Execute each tool call
for tool_call in tool_response['tool_calls']:
    result = await self.tool_registry.execute_tool(
        tool_call['name'],
        **tool_call['arguments']
    )
    
# Add results to hypothesis
metadata["binder_data"]["proposed_peptides"].append({
    "sequence": seq_data["sequence"],
    "source": "computational:bindcraft",
    "rationale": f"Generated by BindCraft with pLDDT={plddt}",
    "tool_metadata": {...}
})
```

**Result**: Hypothesis contains BOTH literature AND computational sequences!

## Implementation Details

### 1. New LLM Interface Method

**File**: `../Jnana/jnana/protognosis/core/llm_interface.py`

**Added `generate_with_tools()` method**:

```python
def generate_with_tools(self, prompt: str, tools: List[Dict],
                       system_prompt: Optional[str] = None,
                       temperature: float = 0.7, max_tokens: int = 1024) -> Dict:
    """
    Generate a response with tool calling support.
    
    Returns:
        Dictionary containing:
            - 'content': Text response (if any)
            - 'tool_calls': List of tool calls (if any)
            - 'finish_reason': Why generation stopped
            - 'prompt_tokens': Number of prompt tokens
            - 'completion_tokens': Number of completion tokens
    """
```

**Implementation for OpenAI** (lines 429-498):

```python
def generate_with_tools(self, prompt: str, tools: List[Dict], ...) -> Dict:
    response = self.client.chat.completions.create(
        model=self.model,
        messages=messages,
        tools=tools,  # Enable function calling
        temperature=temperature,
        max_tokens=max_tokens
    )
    
    # Parse tool calls if present
    tool_calls = []
    if message.tool_calls:
        for tool_call in message.tool_calls:
            tool_calls.append({
                'id': tool_call.id,
                'name': tool_call.function.name,
                'arguments': json.loads(tool_call.function.arguments)
            })
    
    return {
        'content': message.content,
        'tool_calls': tool_calls,
        'finish_reason': finish_reason,
        'prompt_tokens': prompt_tokens,
        'completion_tokens': completion_tokens
    }
```

### 2. Tool Calling Helper Method

**File**: `../Jnana/jnana/protognosis/agents/specialized_agents.py`

**Added `_call_tools_if_needed()` method** (lines 329-411):

```python
async def _call_tools_if_needed(self, hypothesis_response: Dict, 
                                tools: List[Dict], 
                                research_goal: str) -> List[Dict]:
    """
    Two-step approach: Ask LLM if it wants to call tools, then execute them.
    """
    # Extract target sequence
    target_sequence = hypothesis_response.get("binder_data", {}).get("target_sequence", "")
    
    # Create prompt asking if LLM wants to use tools
    tool_decision_prompt = f"""
    You have generated a hypothesis for: {research_goal}
    
    Target protein sequence: {target_sequence[:100]}...
    
    Would you like to use computational design tools to generate additional binders?
    
    Available tools:
    - bindcraft_design: Uses ProteinMPNN to computationally design binders
    """
    
    # Call LLM with tool calling support
    tool_response = self.llm.generate_with_tools(
        prompt=tool_decision_prompt,
        tools=tools,
        system_prompt="You are a computational biologist...",
        temperature=0.3
    )
    
    # Execute tool calls
    results = []
    for tool_call in tool_response.get('tool_calls', []):
        result = await self.tool_registry.execute_tool(
            tool_call['name'],
            **tool_call['arguments']
        )
        results.append(result)
    
    return results
```

### 3. Integration in Hypothesis Generation

**File**: `../Jnana/jnana/protognosis/agents/specialized_agents.py`

**Modified `_generate_hypothesis()` method** (lines 227-259):

```python
# STEP 1: Generate initial hypothesis (structured JSON)
response_data = self.llm.generate_with_json_output(
    prompt, schema, system_prompt=system_prompt
)

# Add binder data if present
if "binder_data" in response:
    metadata["binder_data"] = response["binder_data"]

# STEP 2: Ask LLM if it wants to use tools
if tools and is_binder_design and hasattr(self.llm, 'generate_with_tools'):
    tool_results = await self._call_tools_if_needed(response, tools, research_goal)
    
    if tool_results:
        # STEP 3: Incorporate tool results
        for result in tool_results:
            if result.get("success") and result.get("top_sequences"):
                for seq_data in result["top_sequences"]:
                    metadata["binder_data"]["proposed_peptides"].append({
                        "sequence": seq_data["sequence"],
                        "source": "computational:bindcraft",
                        "rationale": f"Generated by BindCraft...",
                        "tool_metadata": {
                            "plddt": seq_data.get("plddt"),
                            "pae": seq_data.get("pae"),
                            ...
                        }
                    })
        
        metadata["tool_calls_made"] = True
        metadata["tool_call_count"] = len(tool_results)

# Create hypothesis with all data
hypothesis = ResearchHypothesis(
    content=response["hypothesis"]["content"],
    summary=response["hypothesis"]["summary"],
    metadata=metadata
)
```

## Usage Example

See `examples/example_two_step_tool_calling.py` for a complete example.

**Basic usage**:

```python
# Initialize system
system = BinderDesignSystem(
    config_path="config/binder_config.yaml",
    jnana_config_path="../Jnana/config/config.yaml"
)
await system.initialize()

# Generate hypothesis with tool calling
hypothesis = await system.generate_protein_hypothesis(
    research_goal="Design a peptide binder for SARS-CoV-2 spike protein RBD",
    strategy="binder_gen"
)

# Check if tools were called
if hypothesis.metadata.get("tool_calls_made"):
    print(f"✓ LLM called {hypothesis.metadata['tool_call_count']} tools!")
    
    # Hypothesis contains both literature AND computational sequences
    for peptide in hypothesis.binder_data.proposed_peptides:
        if peptide['source'] == "computational:bindcraft":
            print(f"Computational sequence: {peptide['sequence']}")
            print(f"pLDDT: {peptide['tool_metadata']['plddt']}")
```

## Benefits

1. **LLM-Driven Decision Making**: LLM decides when computational design is appropriate
2. **Automatic Integration**: Tool results are automatically incorporated into hypothesis
3. **Metadata Preservation**: All tool metrics (pLDDT, pAE, pTM) are preserved
4. **Backward Compatible**: Existing code continues to work
5. **Flexible**: Can still use explicit calls if needed

---

## Limitations

1. **OpenAI Only**: Currently only implemented for OpenAI LLM provider
   - Other providers will fall back to explicit calls only
   - Can be extended to Anthropic, Gemini, etc.

2. **Single Tool Call**: LLM typically makes one tool call per hypothesis
   - Could be extended to support multiple sequential tool calls

3. **No Iteration**: Tool results are incorporated once
   - Could be extended to allow LLM to iterate based on results

---

## Future Enhancements

### 1. Support for Other LLM Providers

Implement `generate_with_tools()` for:
- Anthropic Claude (supports tool calling)
- Google Gemini (supports function calling)
- Local models via Ollama (limited support)

### 2. Multi-Tool Chaining

Allow LLM to call multiple tools in sequence:
```python
# LLM calls BindCraft
bindcraft_result = execute_tool("bindcraft_design", ...)

# LLM sees result and decides to call AlphaFold
alphafold_result = execute_tool("alphafold_predict", ...)

# LLM sees structure and decides to call MD simulation
md_result = execute_tool("md_simulate", ...)
```

### 3. Iterative Refinement

Allow LLM to iterate based on tool results:
```python
# Initial design
result1 = execute_tool("bindcraft_design", ...)

# LLM sees low pLDDT, adjusts parameters
result2 = execute_tool("bindcraft_design", temperature=0.2, ...)

# LLM satisfied with result
```

### 4. Tool Call History

Track all tool calls for reproducibility:
```python
hypothesis.metadata["tool_call_history"] = [
    {
        "tool": "bindcraft_design",
        "timestamp": "2024-01-15T10:30:00",
        "parameters": {...},
        "result": {...}
    }
]
```
