# Tool Calling Examples

This directory contains examples demonstrating the **LLM tool calling** feature for BindCraft.

## 🎯 What is Tool Calling?

Tool calling allows the **LLM to decide** when and how to use BindCraft during hypothesis generation, rather than you explicitly calling it. This makes the workflow more autonomous and intelligent.

## 🔄 Three Approaches Explained

### Approach 1: Tool Calling (NEW!)
```python
# LLM decides to call BindCraft
hypothesis = await system.generate_protein_hypothesis(
    research_goal="Design binders using computational tools",
    strategy="binder_gen"
)
# Hypothesis may include computationally designed sequences
```

**When to use:**
- ✅ You want LLM to make design decisions
- ✅ You want autonomous workflow
- ✅ You trust LLM to choose parameters

---

### Approach 2: Explicit Call (EXISTING)
```python
# Generate hypothesis
hypothesis = await system.generate_protein_hypothesis(...)

# Explicitly call BindCraft
results = await bindcraft_agent.analyze_hypothesis(
    hypothesis,
    bindcraft_config
)
```

**When to use:**
- ✅ You need precise control over parameters
- ✅ You're running specific optimization protocol
- ✅ You want reproducible results

---

### Approach 3: Hybrid (BEST OF BOTH!)
```python
# LLM may call tools during generation
hypothesis = await system.generate_protein_hypothesis(...)

# ALSO explicitly call for optimization
results = await bindcraft_agent.analyze_hypothesis(
    hypothesis,
    bindcraft_config
)
```

**When to use:**
- ✅ You want LLM reasoning + controlled optimization
- ✅ You're running complex iterative workflows
- ✅ You want to compare LLM vs optimized sequences

## 🔧 Customization

### Encourage Tool Use
Modify the research goal to encourage LLM to use tools:

```python
research_goal = """
Design peptide binders for target X.

Please use computational design tools to generate novel binders.
"""
```

### Discourage Tool Use
Modify the research goal to focus on literature:

```python
research_goal = """
Design peptide binders for target X based on literature.
"""
```

### Control Parameters
When calling explicitly, you control all parameters:

```python
bindcraft_config = {
    "target_sequence": "...",
    "num_rounds": 5,      # Your choice
    "num_seqs": 100,      # Your choice
    "sampling_temp": 0.3  # Your choice
}
```

## ❓ FAQ

### Q: Will the LLM always call the tool?
**A:** No, it's optional. The LLM decides based on the research goal and its reasoning.

### Q: Can I force the LLM to call the tool?
**A:** Not directly, but you can encourage it by mentioning computational design in the research goal.

### Q: Does this break my existing code?
**A:** No! Existing code works unchanged. Tool calling is additive.

### Q: Can I use both tool calling and explicit calls?
**A:** Yes! The hybrid approach uses both.

### Q: How do I know if the tool was called?
**A:** Check the `source` field in `binder_data.proposed_peptides`:
```python
for peptide in hypothesis.binder_data.proposed_peptides:
    if 'computational' in peptide['source']:
        print("Tool was called!")
```

## 🐛 Troubleshooting

### Tool not being called?
1. Check tool registry: `system.tool_registry.list_tools()`
2. Make sure research goal mentions computational design
3. Check logs for "Tools available for LLM"

### Tool call failed?
1. Check logs for error messages
2. Verify `enable_agents=['computational_design']` is set
3. Make sure BindCraft agent initialized successfully

### Existing pipeline broken?
1. This shouldn't happen! Tool calling is backward compatible
2. Check that you're using the latest code
3. Report issue with error logs
